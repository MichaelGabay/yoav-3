# תרגול Power Query ב-Power BI – חיבור ל-PostgreSQL

> התרגול מבוסס על טבלאות שנוצרו בסקריפט `postgres_setup_power_query_exercise.sql`.  
> הרץ קודם את הסקריפט ב-PostgreSQL, ואז טען את הטבלאות ל-Power BI דרך Power Query ובצע את המשימות הבאות.

---

## הכנה – חיבור ל-PostgreSQL וטעינת נתונים

1. ב-Power BI Desktop: **קבל נתונים** → **PostgreSQL database**.
2. הזן **שרת** (Server) ו-**מסד נתונים** (Database), בחר **אופן חיבור** (DirectQuery או Import).
3. בחר את הטבלאות: `regions`, `categories`, `customers`, `products`, `orders`, `order_details`, `raw_sales_log`.
4. לחץ **טען** (או **Transform Data** אם תרצה לערוך לפני הטעינה).

לאחר הטעינה, כל משימה תתבצע ב-**Power Query Editor** על הטבלאות הרלוונטיות.

---

# משימות Power Query (15+)

## משימה 1 – ניקוי רווחים מיותרים (Trim)

**טבלה:** `customers`  
**מטרה:** להסיר רווחים בתחילת ובסוף העמודה `full_name`.

**הוראות:**  
צור עמודה מותאמת אישית או השתמש ב-**Transform → Format → Trim** על `full_name`.

**פתרון (עמודה מותאמת):**
```powerquery
Text.Trim([full_name])
```

**הסבר:**  
בנתונים אמיתיים שמות ולעיתים אימיילים מגיעים עם רווחים מיותרים. Trim מונע חיפוש/סינון שלא עובד (למשל "יוסי כהן" לעומת "  יוסי כהן  ") ומאחד ערכים שזהים מבחינת תוכן.

---

## משימה 2 – אחידות רישום (אות גדולה/קטנה)

**טבלה:** `customers`  
**מטרה:** ליצור עמודה `email_lower` שבה כל האימייל באותיות קטנות.

**פתרון:**
```powerquery
Text.Lower([email])
```

**הסבר:**  
אימיילים לא רגישים לאות גדולה/קטנה, אבל במערכות דאטה השוואה בין "User@Mail.com" ל-"user@mail.com" יכולה להיכשל. אחידות ל-lower (או upper) מונעת כפילויות וטעויות במיזוגים ובסינונים.

---

## משימה 3 – סינון שורות לפי ערך (Filter)

**טבלה:** `customers`  
**מטרה:** להשאיר רק לקוחות ש-`status` שווה ל-`active`.

**הוראות:**  
לחץ על חץ הסינון בעמודה `status` → סמן רק **active** → OK.

**הסבר:**  
סינון שורות הוא פעולה בסיסית ב-Power Query. כאן אנחנו מתמקדים בלקוחות "פעילים" לדוחות ומדדים, בלי לשנות את הנתונים במקור.

---

## משימה 4 – החלפת ערכים null (Fill / Replace)

**טבלה:** `customers`  
**מטרה:** להחליף תאים ריקים ב-`email` במחרוזת `"לא צוין"`.

**הוראות:**  
בעמודה `email`: **Transform → Replace Values**.  
**Value to Find:** (השאר ריק או בחר null לפי הגרסה).  
**Replace With:** `לא צוין`.

**אלטרנטיבה (עמודה מותאמת):**
```powerquery
if [email] = null or [email] = "" then "לא צוין" else [email]
```

**הסבר:**  
ערכים ריקים מקשים על דוחות (טקסט "ריק" בגרפים, בעיות במיזוג). החלפה בערך ברירת מחדל משפרת קריאות ומאפשרת סינון/קבוצה ברורה ל"ללא אימייל".

---

## משימה 5 – יצירת עמודת תאריך – חודש ושנה

**טבלה:** `customers`  
**מטרה:** ליצור עמודה `signup_year_month` בפורמט `"YYYY-MM"` (למשל `2023-01`) מתאריך `signup_date`.

**פתרון:**
```powerquery
if [signup_date] = null then null else Date.ToText([signup_date], "yyyy-MM")
```

**הסבר:**  
קבוצה לפי חודש/שנה נחוצה לדוחות מגמות (כניסות לקוחות, מכירות). פורמט `YYYY-MM` מתאים למיון כרונולוגי ולציר זמן ב-Power BI.

---

## משימה 6 – מיזוג טבלאות (Merge) – לקוחות ואזורים

**מטרה:** לשלב את `customers` עם `regions` כדי לקבל בעמודת `region_name` (או `country_hebrew`) לכל שורת לקוח.

**הוראות:**  
1. בחר את טבלת `customers`.  
2. **Home → Merge Queries**.  
3. בחר טבלה שנייה: `regions`.  
4. התאם עמודות: `customers[region_code]` = `regions[region_code]`.  
5. Join Kind: **Left Outer**.  
6. אשר, ואז **Expand** על העמודה מהטבלה הממוזגת ובחר את העמודות הרצויות (למשל `region_name`, `country_hebrew`).

**הסבר:**  
מיזוג (Join) מחבר מידע מטבלאות שונות. Left Outer משאיר את כל שורות `customers` גם כשלא נמצא אזור (למשל `region_code` ריק או לא קיים ב-`regions`). כך אפשר לדווח לפי אזור בלי לאבד לקוחות.

---

## משימה 7 – הסרת עמודות (Remove Columns)

**טבלה:** `customers` (או כל טבלה אחרת)  
**מטרה:** להסיר עמודות שלא נדרשות לניתוח (למשל `notes` אם הוספת אותה ממיזוג, או עמודה זמנית).

**הוראות:**  
בחר את העמודות → קליק ימני → **Remove Columns**, או **Home → Choose Columns** ובחר רק את מה שצריך.

**הסבר:**  
פחות עמודות = מודל קל יותר וטעינה מהירה יותר. מומלץ להשאיר רק מה שבאמת משמש לדוחות ו-DAX.

---

## משימה 8 – סינון לפי מספר (Filter by condition)

**טבלה:** `products`  
**מטרה:** להשאיר רק מוצרים ש-`unit_price` גדול מ-100.

**הוראות:**  
סינון על `unit_price` → **Number Filters** → **Greater Than** → 100.

**הסבר:**  
סינון לפי תנאי מספרי (מחיר, כמות, משקל) מאפשר להתמקד בפריטים רלוונטיים (למשל מוצרים יקרים, מלאי מעל סף מסוים) ולהוריד רעש מהמודל.

---

## משימה 9 – עמודה מחושבת – רווח ליחידה

**טבלה:** `products`  
**מטרה:** ליצור עמודה `profit_per_unit` = `unit_price` minus `cost_price`.

**פתרון:**
```powerquery
[unit_price] - [cost_price]
```

**הסבר:**  
עמודות מחושבות ב-Power Query מחשבות ערך פעם אחת בזמן הטעינה. מתאים למדדים כמו רווח גולמי, אחוז מרווח, שער המרה – שימושיים גם ל-DAX ולגרפים.

---

## משימה 10 – החלפת ערכים בטקסט (Replace / Standardize)

**טבלה:** `orders`  
**מטרה:** ליצור עמודה `status_standard` שבה כל ערכי `status` באותיות קטנות (lowercase) כדי לאחד "Shipped", "shipped", "SHIPPED".

**פתרון:**
```powerquery
Text.Lower(Text.Trim([status]))
```

**הסבר:**  
כשהמערכת או המשתמשים מזינים סטטוסים ברישום שונה, קבוצה לפי סטטוס מתפצלת לערכים מיותרים. אחידות ל-lower (או לערך קבוע כמו "Shipped") מאחדת את הנתונים לדוחות נכונים.

---

## משימה 11 – סינון שורות עם תאריך ריק

**טבלה:** `orders`  
**מטרה:** להשאיר רק שורות שבהן `order_date` לא ריק (לא null).

**הוראות:**  
סינון על `order_date` → **Remove Empty**.

**הסבר:**  
הזמנות ללא תאריך מקשות על ניתוח לפי זמן. הסרת שורות ריקות (או החלפה בערך ברירת מחדל) מונעת טעויות בחישובי תאריכים ובגרפי זמן.

---

## משימה 12 – פירוק עמודת טקסט (Split Column)

**טבלה:** `raw_sales_log`  
**מטרה:** לפרק את `log_text` לעמודות נפרדות לפי המפריד `|`.

**הוראות:**  
בחר את `log_text` → **Transform → Split Column → By Delimiter**.  
Delimiter: **Custom** → `|`.  
בחר **At each occurrence** או **At left-most/right-most** לפי הצורך. אשר.

**הסבר:**  
נתונים גולמיים מגיעים לעיתים כמחרוזת אחת עם מפרידים (CSV, לוגים). פירוק לעמודות מאפשר שימוש בשדות בנפרד (סוג אירוע, ProductId, Qty, Price) לניתוח ולמיזוג עם טבלאות אחרות.

---

## משימה 13 – חילוץ ערך מתוך טקסט (Extract)

**טבלה:** `raw_sales_log` (אחרי פירוק לוג לפי `|`, או על העמודה המקורית).  
**מטרה:** מתוך שדה שמכיל מחרוזת כמו `"ProductId=1"`, ליצור עמודה `product_id` שמכילה רק את המספר (1).

**פתרון (אם יש עמודה עם "ProductId=1"):**
```powerquery
Number.From(Text.Remove([ColumnName], Text.ToList("ProductId=")))
```
או עם **Extract → Text Between Delimiters**: מתחיל אחרי `=`, עד סוף המחרוזת.

**הסבר:**  
חילוץ מספרים או מחרוזות מתוך שדה טקסט מאפשר המרה לסוג נתונים נכון (מספר, תאריך) ושימוש בשדה במיזוגים, סינונים וחישובים.

---

## משימה 14 – Group By – ספירת הזמנות ללקוח

**טבלה:** `orders`  
**מטרה:** ליצור טבלה חדשה (או query חדש) עם ספירת הזמנות לכל `customer_id`.

**הוראות:**  
1. **Transform → Group By**.  
2. **Group by:** `customer_id`.  
3. **New column name:** `order_count`.  
4. **Operation:** **Count Rows**.  
5. אשר.

**הסבר:**  
Group By מסכם נתונים ברמת קבוצה (למשל לפי לקוח, לפי מוצר, לפי חודש). ספירת שורות נותנת "כמה הזמנות ללקוח" – בסיס ל-KPI, דירוג לקוחות ומדדי נאמנות.

---

## משימה 15 – שינוי סוג נתונים (Data Type)

**טבלאות שונות**  
**מטרה:** לוודא ש-`unit_price` ו-`cost_price` ב-`products` מסוג **Decimal**, ש-`order_date` ב-`orders` מסוג **Date**, ו-`quantity` ב-`order_details` מסוג **Whole Number**.

**הוראות:**  
לחץ על סמל הסוג ליד שם העמודה → בחר את הסוג הרצוי (Decimal, Date, Whole Number וכו').

**הסבר:**  
סוג נתונים נכון חשוב לחישובים (סכומים, ממוצעים) ולציר זמן (תאריכים). אם Power Query מזהה סוג לא נכון (למשל טקסט במקום מספר), יש לשנות ידנית.

---

## משימה 16 – הסרת שורות כפולות (Remove Duplicates)

**טבלה:** `categories`  
**מטרה:** להסיר שורות כפולות (בנתונים יש קטגוריה "אלקטרוניקה" פעמיים).

**הוראות:**  
בחר את העמודות הרלוונטיות (למשל `category_name` או את כל העמודות) → **Home → Remove Rows → Remove Duplicates**.

**הסבר:**  
כפילויות מעוותות ספירות וסכומים. הסרת כפילויות לפי מפתח (למשל שם קטגוריה או id) מנקה את המודל. שים לב: אם בוחרים רק חלק מהעמודות, כפילות נקבעת לפי העמודות שנבחרו.

---

## משימה 17 – Fill Down – מילוי ערכים חסרים מלמעלה

**טבלה:** כל טבלה עם עמודה שיש בה ערכים חוזרים ואז ריקים (למשל אחרי פיבוט או ייבוא).  
**מטרה:** למלא תאים ריקים בערך מהשורה הקודמת.

**הוראות:**  
בחר את העמודה → **Transform → Fill → Down**.

**הסבר:**  
בקבצי Excel או דוחות שמייצגים היררכיה, לעיתים הערך מופיע רק בשורה הראשונה של כל קבוצה. Fill Down "מוריד" את הערך לכל השורות בקבוצה – שימושי לנורמליזציה של טבלאות.

---

## משימה 18 – שינוי סדר עמודות (Reorder)

**מטרה:** לשנות את סדר העמודות בטבלה כך שעמודות מזהות (id) יהיו ראשונות, ואז שמות ורק אז ערכים מספריים.

**הוראות:**  
גרור עמודות בראש הטבלה, או השתמש ב-**Choose Columns** ובחר את העמודות בסדר הרצוי.

**הסבר:**  
סדר עמודות לא משפיע על חישובים, אבל עוזר לניווט ב-Power Query ולביקורת נתונים. מומלץ לסדר בצורה עקבית (מזהים, מימדים, עובדות).

# סיכום והמלצות

| משימה | טכניקה ב-Power Query | שימוש טיפוסי |
|--------|------------------------|----------|
| 1 | Trim | ניקוי רווחים בטקסט |
| 2 | Text.Lower | אחידות רישום |
| 3, 8, 11 | Filter Rows | סינון לפי ערך או תנאי |
| 4 | Replace Values / if | טיפול ב-null |
| 5 | Date.ToText / עמודה מותאמת | תאריכים לדוחות |
| 6 | Merge Queries | חיבור טבלאות (Join) |
| 7 | Remove Columns | צמצום מודל |
| 9 | Add Column (Custom) | חישוב חדש |
| 10 | Text.Lower + Trim | סטנדרטיזציה |
| 12 | Split Column | פירוק טקסט |
| 13 | Extract | חילוץ מתוך טקסט |
| 14 | Group By | צבירה וספירה |
| 15 | Change Type | סוג נתונים נכון |
| 16 | Remove Duplicates | ניקוי כפילויות |
| 17 | Fill Down | מילוי ערכים חסרים |
| 18 | Reorder | ארגון עמודות |

**המלצה:**  
בצע את המשימות על עותקי הטבלאות (Duplicate) או שמור שלב-שלב (למשל עם "Reference" ו-"Duplicate") כדי לשמור גרסאות ביניים. כך אפשר לחזור אחורה אם צריך ולבנות תהליך ETL מסודר לטעינה ל-Power BI.
